Key findings:

  • Bug: Resource ref parsing fails for org-owned resources. The @owner/name@version format is ambiguous because @ is used for both org prefixes and version pinning. All --instruction, --skill, --claude-settings, and --child flags fail with "invalid resource ref" when the owner starts with @.

  • UX: config view outputs raw JSON with credentials_path exposed. Not harmful, but surprising for a user-facing command.
  • UX: tutorial output appends --help usage block at the end, which is redundant for tutorial content.
  • Minor: create instruction --content "# Role\n\n..." stores literal \n instead of newlines. Shell escaping issue, but the CLI could interpret escape sequences.
